### CL-0010 · 2026-06-15 · Industrial Sewing taxonomy pre-freeze audit (DO-NOT-FREEZE-as-is)
- **Approved by:** Kamal (directive: "Final taxonomy audit before prefix freeze … be ruthless … do not preserve the current structure … freeze once and never redesign").
- **Change:** NEW reference doc — `industrial-sewing-taxonomy-audit.md`. Ruthless audit of the 9 XS subcategories + 105 Machine Kinds + coding axes. **Does not modify the registry** — it recommends the corrected taxonomy and gates the prefix freeze.
- **Root finding:** the system classifies the same concept in three places (Subcategory × Machine Kind × Coding Axis). Bed/feed/needle-count/duty/motor/automation are modeled simultaneously as types, kinds, and axes → combinatorial duplication (e.g. "Cylinder-Bed" appears as a Kind under 6 subcategories).
- **Principle established:** classify by what a machine **IS** (stitch class → Product Type, prefixed), not what it **HAS** (bed/feed/needles/duty/motor/automation → attribute facets) or **DOES** (application/automation preset). Litmus: a Product Type forms a distinct stitch unreachable by configuring another type.
- **Verdicts:** ✅ true types = Lockstitch/Overlock/Coverstitch(rename from "Interlock")/Chainstitch. ❌ downgrade to attributes = **XSH Heavy-Duty** (duty class), **XSD Double-Needle** + **XSM Multi-Needle** (needle_count). ❌ **XSS Special** = catch-all → explode (promote Buttonhole/Bartack/Button-Attach/Blindstitch/Zigzag to types; move ultrasonic/heat-seal → Bonding, snap/rivet → Fastening, robotic cell → Production Systems — non-stitch machines do not belong under Sewing). ⚠️ **XSPA Pattern Sewing** = recode (prefix conflict with lockstitch axis `XSPA`) + reframe as Programmable/CNC automation-type.
- **Hierarchy recommendation:** **Machine Kind must NOT be a structural spine level** — it is a derived preset/overlay over (Type × attributes). Spine stays Division→Category→Product Type→Family→Model→SKU→Variant; attributes resolve variants; Kinds power nav/SEO/sales/AI. Add sibling categories (Bonding & Seam-Sealing, Fastening/Press, Production Systems/Cells).
- **Future-growth test:** the two-axis model survives Sewing-Robots/AI/IoT/Vision/CNC (each = a new attribute value, defined once); the current bundled-kind model fails (every trend multiplies across all stitch types — the "Heavy Duty" failure mode repeated).
- **Deliverables (10):** Classification Audit · Product Type Report · Machine Kind Report (all 105 by disposition) · Attribute Report · Config-Axis Report · Final Hierarchy · 8-perspective + visual-first cross-checks · Prefix-Freeze Impact · Future-Growth · **Freeze/Do-Not-Freeze verdict**.
- **Verdict:** 🔴 **DO NOT FREEZE the current structure** (permanent debt on non-recyclable codes). 🟢 Freeze the corrected two-axis model instead (~10 stitch-class types + one shared attribute set + Kinds-as-overlay + 3 sibling categories). Correction is taxonomy-only (~1 week), and the freeze was already blocked on prefix sign-off + baseline (CL-0001) — so fixing now adds no critical-path delay.
- **Affected artifacts updated:** `industrial-sewing-taxonomy-audit.md` (new) · `product-types-master.md` (pre-freeze review banner) · this log.
- **Conflict scan:** **CLEAN.** No codes changed (recommendation only); reinforces the facet-dictionary + visual-first principles; flags (does not yet resolve) the `XSPA` collision + the 12 open CL-0001 conflicts. Resolution lands in a future CL at sign-off.
